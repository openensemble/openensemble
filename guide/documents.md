# Documents & sharing

The **Docs** drawer (sidebar folder icon) is the unified view of every file in your workspace.

## What's in it

- **Uploads** — anything you've dragged into chat or uploaded directly (PDFs, images, CSVs, code).
- **AI images** — every image generated by your image agent.
- **AI videos** — every video from the video generator.
- **Research documents** — long-form research output from `deep_research`.
- **Code projects** — sandboxed projects the coder has worked in. Downloadable as zip.
- **Notes** — a separate tab for free-form personal notes.

Tabs at the top of the drawer filter by kind (All / Photos / Videos / Code).

## Sharing with other users on the same install

Click the share icon on any item, pick which users to share it with, save. Shared files show up in the recipient's Docs drawer with an "uploaded by {name}" label. Sharing is one-way: the original owner can revoke; recipients can't reshare.

You can also share with **everyone** by setting the audience to "Everyone" — useful for house-wide reference docs.

## Sharing outside the install

OpenEnsemble doesn't generate public download URLs by default — sharing is intra-install. If you want a doc on the public internet, download it locally and host it elsewhere. This is deliberate: most installs aren't internet-facing.

## How agents use documents

The `profile_files` skill is the single tool every agent uses to list and read your files: *"Summarise the PDF I uploaded yesterday"* works because the agent can list and open everything in your Docs drawer. (Earlier versions had a separate `shared-docs` skill — that's been folded into `profile_files`. Cross-user shares show up annotated with the sharing user's name.)

Anything you drop into chat — paperclip, paste, drag — now persists with a stable `file_id` (e.g. `documents:doc_abc123`). Earlier versions kept chat uploads only for the immediate turn; they're now first-class files you can reference in later sessions, attach to email by id, or list via `list_profile_files`.

Agents can also *add* to your Docs drawer:
- Image agent → AI images tab
- Video agent → AI videos tab
- Researcher → Research tab
- Coder → Code Projects tab

## Storage

Files live under `users/{userId}/`:
- `docs/` — uploads
- `desktop/images/`, `desktop/videos/` — AI outputs
- `research/` — research docs
- `coder/projects/` — code projects

Backups (see **Backup, restore, auto-update**) include all of these.

## Deleting

Hover any item → trash icon. AI outputs and uploads are deleted from disk; research docs are removed from the index. Code projects can be deleted from the coder agent or via the Docs drawer.
